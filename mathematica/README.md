# Chromesthesia - Wolfram Language Analysis Pipeline

Wolfram Language scripts that process Daniel Caesar audio files and export data (JSON + PNG) for the Chromesthesia web application.

## Prerequisites

- **Wolfram Mathematica v13+** or **Wolfram Engine** (free for developers)
- Audio files in supported formats: WAV, MP3, FLAC, AIFF, M4A, OGG
- Album cover images in JPG or PNG format

## Directory Structure

### Input (you provide)

```
audio/
  raw/                              # Original audio files
    <album-slug>/
      <song-slug>.wav               # e.g. freudian/best-part.wav
  covers/                           # Album cover artwork
    <album-slug>.jpg                # e.g. freudian.jpg
```

The "Won't Live Here" contrast/terrain analysis expects two specific files:

```
audio/raw/<album-slug>/wont-live-here.wav           # Released version
audio/raw/<album-slug>/wont-live-here-unreleased.wav # Unreleased version
```

### Output (generated by scripts)

```
audio/
  normalized/                       # Intermediate normalized WAV files
    <album-slug>/
      <song-slug>.wav
    import-summary.json

public/data/
  atlas/
    manifest.json                   # (you create this - see below)
    <album-slug>/
      palette.json                  # {dominantColors: string[]}
      songs/
        <song-slug>.json            # SpectralData with gradient
        <song-slug>-heatmap.png     # Spectrogram visualization
  contrast/
    released.json                   # VersionData
    unreleased.json                 # VersionData
    diff.json                       # {diff: number[][], maxDiff: number}
    released-heatmap.png
    unreleased-heatmap.png
    diff-heatmap.png
    comparison.png                  # Side-by-side composite
  terrain/
    released-heightmap.json         # {grid, rows, cols, maxHeight, minHeight}
    unreleased-heightmap.json
```

## Running the Pipeline

Run the scripts in order. Each script depends on the output of previous steps.

### From Mathematica

Open each `.wl` file in Mathematica and evaluate, or run from the command line:

### From Command Line (Wolfram Engine)

```bash
cd mathematica/

# Step 1: Import and normalize audio
wolframscript -file 01-audio-import.wl

# Step 2: Spectral analysis (requires step 1)
wolframscript -file 02-spectral-analysis.wl

# Step 3: Extract album colors (independent of steps 1-2)
wolframscript -file 03-color-extraction.wl

# Step 4: Generate heatmap images (requires step 1)
wolframscript -file 04-heatmap-generation.wl

# Step 5: Compute gradient fields (requires step 2)
wolframscript -file 05-gradient-fields.wl

# Step 6: Generate terrain heightmaps (requires step 1)
wolframscript -file 06-terrain-heightmaps.wl

# Step 7: Contrast analysis (requires step 1)
wolframscript -file 07-contrast-analysis.wl
```

### Run All at Once

```bash
cd mathematica/
for f in 0*.wl; do echo "=== Running $f ===" && wolframscript -file "$f"; done
```

## Manifest File

The web app expects a `manifest.json` at `public/data/atlas/manifest.json`. This file is **not** auto-generated; you must create it to define your album/song structure. Example:

```json
{
  "albums": [
    {
      "slug": "freudian",
      "title": "Freudian",
      "year": 2017,
      "coverUrl": "/covers/freudian.jpg",
      "palette": { "dominantColors": [] },
      "songs": [
        {
          "slug": "best-part",
          "title": "Best Part",
          "duration": 219.5,
          "dataUrl": "/data/atlas/freudian/songs/best-part.json",
          "heatmapUrl": "/data/atlas/freudian/songs/best-part-heatmap.png"
        }
      ]
    }
  ]
}
```

After running `03-color-extraction.wl`, update the `palette.dominantColors` arrays with the values from the generated `palette.json` files.

## Configuration

Each script defines configuration variables at the top. Key parameters:

| Variable | Default | Description |
|---|---|---|
| `$targetSampleRate` | 44100 | Audio normalization sample rate |
| `$fftSize` | 2048 | FFT window size for spectral analysis |
| `$hopSize` | 512 | Hop size between FFT windows |
| `$targetGridRows/Cols` | 128 | Spectral data grid dimensions |
| `$gridSize` (terrain) | 256 | Heightmap grid dimensions |
| `$numColors` | 6 | Dominant colors to extract per album |
| `$imageWidth/Height` | 1024x512 | Heatmap export resolution |
| `$silenceThreshold` | 0.01 | RMS threshold for silence trimming |

## Troubleshooting

- **"No audio files found"**: Ensure your audio files follow the `audio/raw/<album-slug>/<song-slug>.<ext>` naming convention.
- **"Import summary not found"**: Run `01-audio-import.wl` before running dependent scripts.
- **Large JSON files**: The 128x128 spectral grids produce ~200KB JSON files. If file sizes are a concern, reduce `$targetGridRows` and `$targetGridCols`.
- **Memory issues**: For very long tracks, increase Mathematica's memory limit or reduce `$fftSize`.
